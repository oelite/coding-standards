## Overview

This document defines the standard templates and formats for all OElite documentation. Every repository in the OElite ecosystem MUST use these templates for business, technical, and user-facing documentation.

These templates are the single source of truth. Project files in `<repo>/.ai/standards/` may extend but never contradict them.

- **Isabella's business validation** (does the deliverable match expectations?)

**This user story is complete when ALL of the following are true:**

- [ ] Code reviewed by Grace (backend) or Felix (frontend)
- [ ] Security review by Maya (if auth/security-related)
- [ ] Performance review by Victor (if performance-critical)
- [ ] UI matches Jonathan's design spec (if frontend)
- [ ] All screenshots captured by Isabella (Screenshot-001, Screenshot-002)
- [ ] User guide updated by Isabella
- [ ] API documentation updated by Isabella (if API changes)
- [ ] Business validation by Isabella (matches BRD/SRS)
- [ ] Deployed to production by Ethan
- [ ] No unresolved risks or blocked items

## 8. Screenshot Standards

- Be captured using Playwright (no manual screenshots or mockups)
- Follow naming convention: `<feature>-<step>.png` (e.g., `login-01.png`, `login-02.png`)
- Include alt text for accessibility
- Be stored in `docs/user-guides/<role>/screenshots/`
- Be updated whenever the UI changes
- Reference the related user story (e.g., "See US-001, Screenshot-001")
