# 0.3 Project Data Schema Template

## Purpose
This template guides Data Architects, Database Administrators, and AI agents in creating precise `.spec/0.3_project_data_schema.md` documents. The resulting document defines the exact data structure requirements, predefined schemas, and entity relationships for implementation. It must reference existing entities or databases where applicable, and provide scaffolding instructions. This document serves as the single source of truth for data structures - when conflicts arise between this schema and code, always verify and update this document first.

## Scaffolding Guidelines

### When Existing Entities Exist
1. **Reference Existing Schema**: Document exact table/collection names and structures
2. **Extension Strategy**: How to add new fields without breaking existing systems
3. **Backward Compatibility**: Ensure changes don't break existing consumers
4. **Migration Path**: Clear steps to transition to new schema

### When Scaffolding New Schema
1. **Stakeholder Consultation**: Confirm requirements with data owners
2. **Sample Data Generation**: Create realistic test data
3. **Validation Scripts**: Automated checks for data integrity
4. **Documentation Updates**: Keep this document current with changes

### Conflict Resolution Process
1. **Identify Discrepancy**: Compare code implementation vs. this document
2. **Root Cause Analysis**: Determine which version is correct
3. **Update Document**: Modify this schema to reflect correct structure
4. **Code Alignment**: Update code to match corrected schema
5. **Testing**: Validate changes don't break functionality
6. **Communication**: Notify team of schema changes
