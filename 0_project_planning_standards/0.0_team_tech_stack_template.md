## Purpose
This template guides Stakeholders, Project Managers, and AI agents in creating detailed `.spec/0.0_skills.md` documents. The resulting document catalogs the current development team's technical skillsets and technology stack proficiencies. This serves as a baseline for generating software requirements and implementation plans, ensuring recommendations align with existing team capabilities and minimize learning curves for new technologies.

## Data Collection Guidelines

### Interview Process
1. **Team Lead Interview**: Gather overall team capabilities
2. **Individual Assessments**: Survey each developer for specific skills
3. **Project Reviews**: Analyze past projects for technology usage
4. **Certification Audits**: Verify official certifications and training

### Assessment Criteria
- **Proficiency Levels**:
  - **Beginner**: Basic understanding, needs guidance
  - **Intermediate**: Can work independently, some mentoring needed
  - **Advanced**: Strong expertise, can mentor others
  - **Expert**: Deep knowledge, architectural decisions

- **Team Coverage**: Percentage of team with working knowledge
- **Years Experience**: Average team experience with technology

### Validation Methods
- **Code Reviews**: Assess actual implementation quality
- **Project Deliverables**: Review completed work samples
- **Knowledge Tests**: Quick assessments for key technologies
- **Peer Reviews**: Team members validate each other's skills

## Usage in Planning

### Software Requirements Generation
1. **Technology Selection**: Prioritize technologies with high team proficiency
2. **Architecture Decisions**: Choose patterns familiar to the team
3. **Timeline Estimation**: Account for learning curves on new technologies
4. **Risk Assessment**: Identify gaps requiring additional training

### Implementation Planning
1. **Task Assignment**: Match tasks to developers with appropriate skills
2. **Training Integration**: Include learning time for new technologies
3. **Mentorship Planning**: Pair experienced with less experienced team members
4. **Quality Gates**: Ensure code reviews by subject matter experts

### Continuous Improvement
- **Quarterly Updates**: Reassess team skills and technology preferences
- **Training Tracking**: Monitor completion of planned skill development
- **Project Retrospectives**: Capture lessons learned and skill gaps discovered
- **Technology Radar**: Maintain awareness of relevant technology trends

- **Update Frequency**: Quarterly review and updates
